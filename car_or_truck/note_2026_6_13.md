# 第二天学习笔记：从线性可分到重叠数据，探索过拟合与正则化

## 实验目标

在第一天成功训练线性可分 Car/Truck 分类器的基础上，今天的目标是引入更真实的数据特性——**特征重叠**，并观察模型的过拟合现象，同时学习如何通过**正则化、早停、交叉验证**和**增加数据量**来提升泛化能力。

我们编写了两个 Python 脚本：
- `car_or_truck_overlap.py`：使用 200 个样本（每类 100），简单的双层网络（8→8 神经元），无 dropout，训练 500 轮。
- `car_or_truck_overlap_op.py`：使用 1000 个样本（每类 500），更深的网络（16→16 神经元），加入 Dropout(0.2) 和权重衰减（weight_decay=1e-4），并采用早停和交叉验证。

两张热力图分别展示了两个模型在特征空间上的决策边界。

## 第一部分：重叠数据的生成

为了模拟真实场景，我们让 Car 和 Truck 在体积和重量上都有重叠区间：

| 类别 | 体积范围 | 重量范围 |
| :--- | :--- | :--- |
| Car   | 0.5 - 2.3 | 0.5 - 2.0 |
| Truck | 2.0 - 5.0 | 1.5 - 4.0 |

重叠区域：体积 [2.0, 2.3]，重量 [1.5, 2.0]。这个区域内两类样本共存，无法用一条直线完美分开。

关键代码（以 `_op.py` 为例，每类 500 样本）：
```python
car_volume = torch.rand(500, 1) * 1.8 + 0.5   # 0.5-2.3
car_weight = torch.rand(500, 1) * 1.5 + 0.5   # 0.5-2.0
truck_volume = torch.rand(500, 1) * 3.0 + 2.0 # 2.0-5.0
truck_weight = torch.rand(500, 1) * 2.5 + 1.5 # 1.5-4.0
```

## 第二部分：两个模型的对比

### 模型1：`car_or_truck_overlap.py`（200样本，无正则化）

**结构**：`Linear(2,8) → ReLU → Linear(8,8) → ReLU → Linear(8,1) → Sigmoid`  
**优化器**：Adam(lr=0.01)，无 weight decay  
**训练**：固定 500 轮，无早停  
**数据量**：200 个样本

**关键输出**（500 轮后）：
- 训练准确率 99%，测试准确率 100%（但测试集仅 40 个样本）
- 重叠区域样本 (2.1, 1.6) 预测为 Truck，概率 0.876（非常确信）
- (1.8, 2.2) 重量超出 Car 上限，却被判为 Car（概率 0.003）
- 损失在 1400 轮后降为 0，但出现过拟合迹象

**问题**：模型容量相对数据量过大，缺乏正则化，导致过拟合——虽然训练集表现完美，但对微小扰动敏感，且无法表达不确定性。

### 模型2：`car_or_truck_overlap_op.py`（1000样本，带正则化与早停）

**结构**：`Linear(2,16) → ReLU → Dropout(0.2) → Linear(16,16) → ReLU → Dropout(0.2) → Linear(16,1) → Sigmoid`  
**优化器**：Adam(lr=0.01, weight_decay=1e-4)  
**训练**：早停（patience=20），最大 500 轮，实际在第 45 轮停止  
**数据量**：1000 个样本

**关键输出**：
- 第 0 轮验证准确率已达 95.63%，早停于 45 轮。
- 测试准确率 100%（200 个测试样本）。
- 5 折交叉验证平均准确率 **98.75% ± 0.40%**，稳定性极高。
- 重叠区域样本 (2.1, 1.6) 预测为 **Car**，概率 0.016（强烈偏向 Car，因为重叠区域中 Car 样本密度更高）。
- (1.8, 2.2) 预测为 Car，概率 0.158（仍偏向 Car，但不如之前极端）。

## 第三部分：关键概念与改进措施总结

### 过拟合（Overfitting）

**现象**：模型在训练集上损失趋近于 0，准确率 100%，但在验证集或新样本上表现不稳定，决策边界扭曲。  
**原因**：模型容量（参数数量）相对于训练数据量过大，或者训练时间过长。  
**本例对比**：200 样本 + 无正则化 → 过拟合；1000 样本 + 正则化 + 早停 → 泛化良好。

### 早停（Early Stopping）

监控验证损失（或验证准确率），当验证损失连续多个 epoch 不再下降时停止训练，并恢复最佳模型参数。  
在我们的代码中，`patience=20`，验证损失连续 20 轮未改善即停止。这避免了因训练过度而记忆噪声。

### Dropout

在训练过程中随机“丢弃”一定比例的神经元输出（此处设为 0.2）。Dropout 强制网络不依赖特定神经元，从而学习更鲁棒的特征，是一种有效的正则化手段。

### 权重衰减（Weight Decay）

在损失函数中添加权重的 L2 范数作为惩罚项，等价于每次参数更新时缩小权重值。它限制模型参数过大，降低过拟合风险。我们在 Adam 优化器中设置 `weight_decay=1e-4`。

### 交叉验证（K-Fold Cross Validation）

将训练+验证集划分为 K 份，轮流将其中一份作为验证集，其余 K-1 份作为训练集，最后取平均准确率。它提供了对模型泛化性能的更可靠估计，避免了单次随机划分的偶然性。  
本例中 5 折交叉验证平均准确率 98.75%，标准差仅 0.40%，证明模型稳定。

### 数据量的重要性

从 200 样本增加到 1000 样本，模型从过拟合变为良好泛化。更多数据能让模型学到真实的分布规律，而不是个别点的噪声。
