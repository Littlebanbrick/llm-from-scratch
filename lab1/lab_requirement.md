## Lab: MNIST 手写数字识别与超参数探索

你已经完成了 freeCodeCamp 的 deep learning crash course，掌握了神经网络的基本概念（前向传播、反向传播、梯度下降、过拟合、正则化等）。在这个 Lab 中，你将把所学应用到经典的 **MNIST 手写数字识别** 任务上，并系统地探索不同超参数（隐藏层数、神经元个数、激活函数、Dropout 等）对模型性能的影响。通过实验，你将直观地理解“模型容量”与“泛化能力”之间的权衡。

### 实验目标

- 构建一个全连接神经网络（MLP）对 MNIST 的 10 个数字（0~9）进行分类。
- 理解并记录不同超参数配置下的训练/验证准确率、损失曲线。
- 分析为什么某些配置会导致过拟合或欠拟合。

### 所需工具

- Python 3.8+，PyTorch，torchvision（用于下载 MNIST），matplotlib，numpy。
- 如果未安装 `torchvision`，请运行：`pip install torchvision`

### 数据准备

MNIST 数据集包含 60,000 张 28×28 的灰度手写数字图像，以及 10,000 张测试图像。每张图像的像素值在 0~255 之间。我们将图像展平为 784 维向量，并将像素值归一化到 [0,1] 区间。

### 任务列表

1. **实现一个可配置的多层感知机（MLP）**，支持设置隐藏层数量、每层神经元数量、激活函数类型、是否使用 Dropout。
2. **训练基础模型**（建议：2 个隐藏层，每层 128 个神经元，ReLU 激活，无 Dropout）作为基准。
3. **进行一系列对比实验**（见下表），记录每个实验的最终验证准确率以及训练/验证损失曲线。
4. **撰写简短实验报告**：解释观察到的现象，回答思考题。

## 探索任务

请基于基准代码，逐一修改超参数，重新运行训练，并记录结果。每个实验除了记录最终验证准确率和测试准确率外，还要**对比损失曲线的形状**（是否平滑、是否出现过拟合迹象）。

### 实验 A：改变隐藏层数量

保持每层神经元数为 128，无 Dropout，分别尝试：

- `hidden_layers = []` (无隐藏层，即逻辑回归)
- `hidden_layers = [128]` (1 个隐藏层)
- `hidden_layers = [128, 128]` (2 个隐藏层，基准)
- `hidden_layers = [128, 128, 128]` (3 个隐藏层)

**观察**：随着层数增加，训练准确率如何变化？验证准确率是否出现过拟合（验证损失先下降后上升）？为什么？

### 实验 B：改变神经元数量

固定 2 个隐藏层，尝试不同宽度：

- `hidden_layers = [16, 16]`
- `hidden_layers = [64, 64]`
- `hidden_layers = [128, 128]` (基准)
- `hidden_layers = [512, 512]`

**观察**：当神经元过少（16）时，模型是否欠拟合（训练和验证准确率都低）？神经元过多（512）时，是否出现过拟合（训练准确率接近 100%，验证准确率反而下降）？解释容量与泛化的关系。

### 实验 C：激活函数对比

固定 2 个 128 隐藏层，分别使用：

- `activation = 'relu'`
- `activation = 'tanh'`
- `activation = 'sigmoid'`

**观察**：哪个激活函数收敛最快？哪个可能遇到梯度消失（尤其是深层时）？为什么 ReLU 现在是主流？

### 实验 D：Dropout 的影响

固定 2 个 128 隐藏层，ReLU 激活，尝试：

- `dropout = 0.0` (基准)
- `dropout = 0.2`
- `dropout = 0.5`

**观察**：Dropout 是否降低了训练准确率但提升了验证/测试准确率？是否使损失曲线更平滑？尝试绘制训练损失和验证损失曲线，观察过拟合是否减轻。

### 实验 E：Batch Normalization

在实验 D 的最佳 Dropout 配置基础上，添加 `use_batch_norm=True`，观察对收敛速度和准确率的影响。
